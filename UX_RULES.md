# CHAY UX Rules

## Dashboard Hierarchy

RUN MODE is always the dominant focal point.

The first screen should prioritize:

1. identity cue
2. emotional nudge
3. timer/session entry
4. one clear action button

Secondary information must support starting, not compete with it. Momentum, history, templates, and settings should feel quiet and compressed.

## Motion Philosophy

Motion should feel grounded, intentional, and calm.

Use motion for:

- entering RUN MODE
- completing a session
- acknowledging a tap
- reinforcing momentum
- guiding attention during state changes

Avoid:

- flashy startup animation
- constant decorative movement
- bouncy gamified effects
- motion that delays starting

Every animation should answer: "What changed emotionally or functionally?"

## Typography Rules

Typography is bold for identity, quiet for support.

- Hero type can be large and forceful.
- Tool panels use compact headings.
- Supporting copy must be short.
- Avoid long explanatory paragraphs on the dashboard.
- Do not shrink text with viewport-width formulas.
- Keep letter spacing at `0`.
- Text must never overflow buttons, cards, or mobile containers.

## Copywriting Tone

CHAY speaks like a calm execution partner.

Use copy that is:

- direct
- grounded
- emotionally supportive
- action-oriented
- sparse

Good:

- "Bắt đầu lại một nhịp mới."
- "Một phiên là đủ để bắt đầu."
- "Không cần hoàn hảo. Chỉ cần tiếp tục chạy."
- "M đã giữ nhịp."

Avoid:

- hype
- shame
- hustle language
- over-motivation
- achievement spam
- long instructions

## Interaction Principles

Every interaction should acknowledge the user lightly.

Use:

- subtle hover lift
- tap compression
- soft glow on active state
- smooth overlay transitions
- delayed completion actions for emotional pause

Do not add interaction complexity. A tactile feeling is good; more controls are not.

## Mobile-First Constraints

Mobile is the primary stress test.

On mobile:

- first screen must make users want to start
- RUN MODE stays visually dominant
- hide or compress secondary systems
- avoid dense grids
- keep nav minimal
- keep buttons large enough to tap
- never require horizontal scrolling in the app

If a feature feels heavy on mobile, simplify it before shipping.

## Focus-State Principles

Focus mode should feel like crossing a threshold.

It should:

- dim the dashboard
- reduce surrounding noise
- center the timer
- keep the commitment visible
- offer only essential actions

Completion should feel rewarding but not loud:

- pulse/glow lightly
- pause before showing actions
- reinforce output and momentum
- encourage returning tomorrow without pressure

Focus mode is sacred. Do not turn it into a dashboard.
